文件流基础

发表于2025-10-29|更新于2025-11-06|C++8. 文件IO
|浏览量:

核心要点速览

  • 三类文件流：ifstream（读）、ofstream（写）、fstream（读写），分别继承自istream、ostream、iostream。
  • 打开模式：in（读）、out（写，默认截断）、app（追加）、ate（定位到尾）、binary（二进制）。
  • 操作流程：打开（构造 /open()）→ 检查（is_open()）→ 读写 → 关闭（close()）。
  • 状态判断：eof()（文件尾）、fail()（可恢复错误）、bad()（严重错误）、good()（无错）。

一、文件流类（分工与默认行为）

类名 功能 默认模式 继承方法（复用性）
ifstream 仅读文件 ios::in >>、getline()等输入方法
ofstream 仅写文件 ios::out（含trunc） <<、put()等输出方法
fstream 读写文件 无（需显式指定） 兼具ifstream和ofstream方法

二、打开模式

模式 作用 典型场景 易错点
ios::in 读权限，文件不存在则失败 读取配置文件 ifstream默认，无需显式加
ios::out 写权限，默认清空文件（trunc） 覆盖写入新内容 加app可取消截断，改为追加
ios::app 仅在文件末尾追加写入 日志文件追加 指针强制固定在末尾，无法移动
ios::ate 打开后指针到末尾，可移动 读写时先定位到文件尾 区别于app，可自由移动指针
ios::binary 二进制模式，禁用换行符转换 读写图片、结构体 必须显式指定，否则为文本模式

三、操作流程

  1. 打开：ifstream ifs("a.txt", ios::in); 或 ofstream ofs; ofs.open("b.txt");
  2. 检查：if (!ifs.is_open()) { /* 处理打开失败 */ }（必须步骤）
  3. 读写：
    • 文本：ifs >> val;、getline(ifs, str);；ofs << "data";
    • 二进制：ifs.read(reinterpret_cast<char*>(&obj), sizeof(obj));；ofs.write(...)
  4. 关闭：ifs.close();（显式关闭，刷新缓冲区）

四、状态判断函数

函数 含义 典型触发场景
eof() 到达文件末尾 读操作完成后
fail() 可恢复错误（如格式错） 用>>读整数时遇到字母
bad() 严重错误（如文件损坏） 磁盘故障导致读写失败
good() 操作无错 读写正常完成后

五、问答

1. ios::app和ios::ate的核心区别？

app强制只能在文件末尾写，指针不可移动；ate打开后指针在末尾，但可通过seekg/seekp移动到任意位置读写。

2. 二进制与文本文件操作的本质区别？

二进制用read()/write()+ios::binary，无换行符转换；文本用>>/<<，会转换换行符（如\n→\r\n）。
